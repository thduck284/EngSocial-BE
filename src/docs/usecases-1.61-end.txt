========================================================================================
  UC-1.61 — Thống kê học tập, thành tựu và hồ sơ kỹ năng
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.61 |
| Tên Use Case                                           | Thống kê học tập, thành tựu và hồ sơ kỹ năng |
| Mô tả                                                  | Cho phép người dùng đã đăng nhập xem thống kê cấp độ và điểm kinh nghiệm, danh sách thành tựu (đã mở khóa hoặc chưa), |
|                                                        | xem và (tuỳ chính sách) cập nhật biểu đồ điểm kỹ năng đọc/nghe/viết/… trên máy chủ. |
| Đối tượng                                              | Người dùng (đã đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng đã đăng nhập. |
|                                                        | 2.Người dùng mở trang “Thống kê”, “Thành tựu” hoặc “Kỹ năng của tôi”. |
| Hậu điều kiện                                          | 1.Số liệu thống kê, thành tựu và (nếu có) biểu đồ kỹ năng hiển thị đúng theo phản hồi thành công từ máy chủ. |
|                                                        | 2.Sau khi cập nhật hồ sơ kỹ năng thành công, giao diện phản ánh dữ liệu mới. |
| Luồng cơ bản                                           | 1.Người dùng mở trang thống kê cá nhân. |
|                                                        | 2.Hệ thống (Frontend) gửi yêu cầu lấy thống kê (cấp, XP, streak, …) đến máy chủ. |
|                                                        | 3.Hệ thống (Backend) trả về dữ liệu; Hệ thống hiển thị cho người dùng. |
|                                                        | 4.Người dùng mở trang thành tựu. |
|                                                        | 5.Hệ thống (Frontend) gửi yêu cầu lấy danh sách thành tựu đến máy chủ. |
|                                                        | 6.Hệ thống (Backend) trả về danh sách; Hệ thống hiển thị trạng thái từng huy hiệu. |
|                                                        | 7.Người dùng mở trang hồ sơ kỹ năng (biểu đồ). |
|                                                        | 8.Hệ thống (Frontend) gửi yêu cầu lấy dữ liệu kỹ năng đến máy chủ; Hệ thống hiển thị biểu đồ. |
|                                                        | 9.(Tuỳ chọn) Người dùng chỉnh các giá trị được phép sửa và nhấn lưu. |
|                                                        | 10.Hệ thống (Frontend) gửi yêu cầu cập nhật hồ sơ kỹ năng đến máy chủ. |
|                                                        | 11.Hệ thống (Backend) lưu và trả kết quả; Hệ thống cập nhật biểu đồ. |
| Luồng thay thế                                         | Không có. |
| Luồng ngoại lệ                                         | (2a) Không tải được thống kê: |
|                                                        |   2a1.Hệ thống hiển thị thông báo lỗi. |
|                                                        |   2a2.Người dùng thử tải lại. |
|                                                        | (5a) Không tải được thành tựu: Hệ thống hiển thị thông báo lỗi. |
|                                                        | (8a) Không tải hoặc không lưu được hồ sơ kỹ năng: Hệ thống hiển thị thông báo lỗi theo máy chủ. |
|                                                        | (E1) Lỗi hệ thống hoặc mạng: Hệ thống hiển thị thông báo lỗi nội bộ hoặc lỗi kết nối; nếu máy chủ trả 401 thì xử lý |
|                                                        |   theo luồng chung của ứng dụng (ví dụ yêu cầu đăng nhập lại). |

========================================================================================
  UC-1.62 — Chặn người dùng
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.62 |
| Tên Use Case                                           | Chặn người dùng |
| Mô tả                                                  | Cho phép người dùng đã đăng nhập chặn hoặc bỏ chặn một người dùng khác (ví dụ từ hồ sơ hoặc sau tương tác xấu). Tuỳ chọn ngôn ngữ, chủ đề và thông báo: xem UC-1.6. |
| Đối tượng                                              | Người dùng (đã đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng đã đăng nhập. |
|                                                        | 2.Giao diện cung cấp thao tác chặn/bỏ chặn trên đối tượng cụ thể. |
| Hậu điều kiện                                          | Sau khi chặn hoặc bỏ chặn thành công, danh sách tương tác (tin nhắn, bạn bè, …) tuân theo chính sách máy chủ. |
| Luồng cơ bản                                           | 1.Người dùng chọn “Chặn” đối với một người dùng khác. |
|                                                        | 2.Hệ thống (Frontend) gửi yêu cầu chặn đến máy chủ; Hệ thống hiển thị xác nhận. |
|                                                        | 3.(Tuỳ chọn) Người dùng chọn “Bỏ chặn”; Hệ thống (Frontend) gửi yêu cầu bỏ chặn đến máy chủ và cập nhật giao diện. |
| Luồng thay thế                                         | Không có. |
| Luồng ngoại lệ                                         | (2a) Chặn hoặc bỏ chặn thất bại (không tồn tại, không đủ quyền, …): Hệ thống hiển thị thông báo theo máy chủ. |
|                                                        | (E1) Lỗi hệ thống hoặc mạng: Hệ thống hiển thị thông báo lỗi nội bộ hoặc lỗi kết nối; nếu máy chủ trả 401 thì xử lý |
|                                                        |   theo luồng chung của ứng dụng (ví dụ yêu cầu đăng nhập lại). |
