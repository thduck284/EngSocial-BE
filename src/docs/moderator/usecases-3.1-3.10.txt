  UC-3.1 — Tra cứu danh sách bài học
================================================================================
| Thuộc tính       | Nội dung |
|------------------|-----------------------------------------------------------------------------|
| Mã Use Case      | UC-3.1 |
| Tên Use Case     | Tra cứu danh sách bài học |
| Mô tả            | Moderator lấy và xem danh sách bài học; lọc theo kỹ năng, cấp độ, chủ đề; tìm theo từ khóa; sắp xếp và phân trang. Không bao gồm tạo/sửa/xóa bài. |
| Đối tượng        | Moderator |
| Tiền điều kiện   | Đã đăng nhập với quyền moderator. |
| Hậu điều kiện    | Danh sách hiển thị khớp dữ liệu máy chủ tại thời điểm tra cứu. |
| Luồng cơ bản     | 1.Moderator yêu cầu tra cứu danh sách bài học (kèm điều kiện chỉ lấy loại bài học, không gồm bài luyện). |
|                  | 2.Hệ thống kiểm tra phiên và quyền moderator. |
|                  | 3.Hệ thống trả về danh sách hoặc thông báo lỗi. |
|                  | 4.Moderator xem bảng kết quả; có thể đổi bộ lọc, từ khóa, chiều sắp xếp, trang (phần lớn xử lý trên dữ liệu đã tải theo triển khai hiện tại). |
| Luồng thay thế   | Không có. |
| Luồng ngoại lệ   | (3a) Lỗi tải: thông báo lỗi; hết phiên thì xử lý theo luồng chung ứng dụng. |

| Luồng ngoại lệ   | (3a) Xóa thất bại: báo lỗi; không xóa dữ liệu. |

  UC-3.5 — Tạo mới bài học
================================================================================
| Thuộc tính       | Nội dung |
|------------------|-----------------------------------------------------------------------------|
| Mã Use Case      | UC-3.5 |
| Tên Use Case     | Tạo mới bài học |
| Mô tả            | Moderator nhập đầy đủ thông tin và nội dung bài học mới, tuỳ chọn đính kèm ảnh đại diện hoặc học liệu, rồi lưu. |
| Đối tượng        | Moderator |
| Tiền điều kiện   | Đã đăng nhập với quyền moderator. |
| Hậu điều kiện    | Bản ghi bài học mới tồn tại trên máy chủ nếu lưu thành công. |
| Luồng cơ bản     | 1.Moderator mở chức năng tạo bài học mới. |
|                  | 2.Hệ thống hiển thị form với giá trị mặc định loại bài học. |
|                  | 3.Moderator nhập thông tin chung (tiêu đề, mô tả, cấp độ, chủ đề, thời lượng, XP thưởng, …) và nội dung bài (câu hỏi, đáp án, nghe/đọc/viết tuỳ loại). |
|                  | 4.(Tuỳ chọn) Moderator tải ảnh hoặc tệp học liệu: hệ thống lưu tệp, trả đường dẫn/mã định danh để gán vào bài. |
|                  | 5.Moderator bấm lưu; hệ thống kiểm tra phía client các trường bắt buộc cơ bản. |
|                  | 6.Hệ thống gửi yêu cầu tạo mới; backend kiểm tra quyền, validate, ghi CSDL. |
|                  | 7.Hệ thống thông báo thành công hoặc lỗi. |
| Luồng thay thế   | (3a) Nhập nhanh từ vựng/câu hỏi hoặc dùng modal hỗ trợ (theo triển khai). |
| Luồng ngoại lệ   | (4a) Tải tệp lỗi: báo lỗi; không gán được ảnh/học liệu. (6a) Dữ liệu không hợp lệ hoặc từ chối quyền: báo theo phản hồi máy chủ. |

================================================================================
| Thuộc tính       | Nội dung |
|------------------|-----------------------------------------------------------------------------|
| Mã Use Case      | UC-3.10 |
| Tên Use Case     | Chấm điểm phần writing của học viên |

|                  | 5.Hệ thống cập nhật hiển thị danh sách kết quả hoặc trạng thái dòng tương ứng. |

| Luồng ngoại lệ   | (4a) Lưu thất bại: giữ form, báo lỗi theo máy chủ. |
