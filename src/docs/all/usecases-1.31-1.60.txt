| Mô tả                                                  | Cho phép người dùng đã đăng nhập thả cảm xúc (reaction) lên bài viết; máy chủ lưu và trả về trạng thái hiển thị. |
| Đối tượng                                              | Người dùng (đã đăng nhập) |

|                                                        | 2.Hệ thống (Frontend) gửi yêu cầu ghi nhận reaction đến máy chủ kèm định danh bài và loại cảm xúc. |

  UC-2.30 — Xem bình luận bài viết
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-2.30 |
| Tên Use Case                                           | Xem bình luận bài viết |
| Mô tả                                                  | Cho phép người dùng đã đăng nhập xem danh sách bình luận của một bài viết; dữ liệu do máy chủ trả về (có thể phân trang). |
| Đối tượng                                              | Người dùng (đã đăng nhập nếu thao tác ghi) |
| Tiền điều kiện                                         | 1.Người dùng đã đăng nhập khi chính sách yêu cầu để xem hoặc tải bình luận. |
|                                                        | 2.Người dùng đang xem chi tiết bài viết và mở khu vực bình luận. |
| Hậu điều kiện                                          | Danh sách bình luận (trang đầu hoặc trang tiếp theo) hiển thị đúng theo phản hồi thành công từ máy chủ. |
| Luồng cơ bản                                           | 1.Người dùng mở tab hoặc khu vực “Bình luận” trên trang chi tiết bài viết. |
|                                                        | 2.Hệ thống (Frontend) gửi yêu cầu lấy danh sách bình luận đến máy chủ (kèm phân trang nếu có). |
|                                                        | 3.Hệ thống (Backend) trả về danh sách bình luận. |
|                                                        | 4.Hệ thống hiển thị các bình luận cho người dùng. |
|                                                        | 5.(Tuỳ thiết kế) Người dùng cuộn tới cuối hoặc chọn “Xem thêm”; Hệ thống (Frontend) gửi yêu cầu lấy trang bình luận tiếp theo. |
|                                                        | 6.Hệ thống (Backend) trả về dữ liệu; Hệ thống nối thêm hoặc cập nhật danh sách. |
| Luồng thay thế                                         | Không có. |
| Luồng ngoại lệ                                         | (2a) Không tải được bình luận lần đầu: Hệ thống hiển thị thông báo lỗi; người dùng thử tải lại. |
|                                                        | (5a) Không tải được trang tiếp theo: Hệ thống hiển thị thông báo lỗi; các bình luận đã hiển thị giữ nguyên. |
|                                                        | (E1) Lỗi hệ thống hoặc mạng; 401: xử lý theo luồng chung ứng dụng. |

========================================================================================
  UC-2.34 — Tin nhắn
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-2.34 |
| Tên Use Case                                           | Nhắn tin (hội thoại) |
| Mô tả                                                  | Cho phép người dùng đã đăng nhập dùng khu tin nhắn: xem danh sách cuộc trò chuyện (chat một-một hoặc nhóm tuỳ sản phẩm), mở luồng tin, gửi tin văn bản và (tuỳ thiết kế) tệp đính kèm, |
|                                                        | xem số tin chưa đọc hoặc tổng chưa đọc, đánh dấu đã đọc khi người dùng xem hội thoại; các thao tác ghi đồng bộ qua máy chủ. Các chức năng mở rộng (chuyển tiếp, sửa/thu hồi tin, cảm xúc trên tin, tạo nhóm, cài đặt thông báo theo hội thoại) tuỳ EngSocial-FE nhưng cùng nguyên tắc gọi máy chủ. |
| Đối tượng                                              | Người dùng (đã đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng đã đăng nhập. |
|                                                        | 2.Người dùng truy cập trang hoặc biểu tượng tin nhắn trên ứng dụng. |
| Hậu điều kiện                                          | 1.Danh sách hội thoại và luồng tin hiển thị đúng theo phản hồi thành công từ máy chủ. |
|                                                        | 2.Số tin chưa đọc và trạng thái “đã đọc” phản ánh sau khi máy chủ xác nhận. |
|                                                        | 3.Sau khi gửi tin thành công, tin xuất hiện trong luồng hội thoại và đối phương có thể nhận theo cơ chế realtime/polling của hệ thống. |
| Luồng cơ bản                                           | 1.Người dùng mở danh sách hội thoại. |
|                                                        | 2.Hệ thống (Frontend) gửi yêu cầu lấy danh sách cuộc trò chuyện đến máy chủ (có thể kèm lấy tổng chưa đọc nếu giao diện hiển thị). |
|                                                        | 3.Hệ thống (Backend) trả dữ liệu; Hệ thống hiển thị danh sách và huy hiệu chưa đọc (nếu có). |
|                                                        | 4.Người dùng chọn một hội thoại. |
|                                                        | 5.Hệ thống (Frontend) gửi yêu cầu lấy lịch sử tin nhắn của hội thoại đó đến máy chủ (có thể phân trang). |
|                                                        | 6.Hệ thống (Backend) trả về tin nhắn; Hệ thống hiển thị luồng hội thoại. |
|                                                        | 7.Hệ thống (Frontend) gửi yêu cầu đánh dấu đã đọc khi người dùng đã xem hội thoại (tuỳ chính sách sản phẩm). |
|                                                        | 8.Người dùng soạn nội dung và (tuỳ chọn) đính kèm tệp, rồi gửi. |
|                                                        | 9.Hệ thống (Frontend) gửi tin lên máy chủ; Hệ thống (Backend) lưu và trả về tin đã ghi nhận. |
|                                                        | 10.Hệ thống cập nhật luồng tin, hàng đợi gửi/nhận và trạng thái đã đọc trên giao diện. |
| Luồng thay thế                                         | (1a) Người dùng bắt đầu chat với một người chưa có hội thoại: Hệ thống (Frontend) yêu cầu máy chủ tạo hoặc lấy hội thoại với người đó rồi chuyển vào bước 5. |
|                                                        | (1b) Người dùng tạo nhóm chat (nếu sản phẩm hỗ trợ): Hệ thống (Frontend) gửi yêu cầu tạo nhóm đến máy chủ; sau khi thành công, Hệ thống mở hội thoại nhóm mới. |
| Luồng ngoại lệ                                         | (2a) Không lấy được danh sách hội thoại: |
|                                                        |   2a1.Hệ thống hiển thị thông báo lỗi. |
|                                                        |   2a2.Người dùng thử tải lại. |
|                                                        | (5a) Không lấy được tin nhắn: Hệ thống hiển thị thông báo lỗi. |
|                                                        | (7a) Đánh dấu đã đọc thất bại: Hệ thống vẫn hiển thị tin nhưng có thể giữ huy hiệu chưa đọc; người dùng thử lại sau. |
|                                                        | (9a) Gửi tin hoặc tệp thất bại (dung lượng, định dạng, quyền, …): Hệ thống hiển thị thông báo theo máy chủ. |
|                                                        | (E1) Lỗi hệ thống hoặc mạng: Hệ thống hiển thị thông báo lỗi nội bộ hoặc lỗi kết nối; nếu máy chủ trả 401 thì xử lý theo luồng chung của ứng dụng (ví dụ yêu cầu đăng nhập lại). |
