========================================================================================
  UC-1.1 — Đăng nhập
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.1 |
| Tên Use Case                                           | Đăng nhập |
| Mô tả                                                  | Cho phép người dùng đăng nhập vào hệ thống bằng email và mật khẩu. Có thể ghi nhớ thông tin |
|                                                        | trên máy nếu người dùng chọn. |
| Đối tượng                                              | Người dùng (chưa đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng đã có tài khoản. |
|                                                        | 2.Người dùng đang ở trang “Sign In”. |
| Hậu điều kiện                                          | 1.Hệ thống chuyển hướng người dùng đến trang “Home” hoặc trang trước đó (nếu hệ thống đã nhớ). |
|                                                        | 2.Nếu người dùng chọn “Ghi nhớ”: Hệ thống lưu email và mật khẩu trên máy để điền sẵn lần sau; nếu không chọn |
|                                                        |   thì xóa để tránh người khác dùng chung máy thấy mật khẩu. |
| Luồng cơ bản                                           | 1.Người dùng truy cập vào trang “Sign In”. |
|                                                        | 2.Hệ thống hiển thị form đăng nhập gồm: Email, Mật khẩu, ô “Ghi nhớ”, liên kết “Forgot Password”, |
|                                                        |   nút đăng nhập bằng Google và Facebook. |
|                                                        | 3.Người dùng nhập thông tin đăng nhập vào form. |
|                                                        | 4.Người dùng nhấn nút “Sign In”. |
|                                                        | 5.Hệ thống (Frontend) kiểm tra tính hợp lệ của dữ liệu. |
|                                                        | 6.Hệ thống (Backend) kiểm tra email và mật khẩu đã đúng chưa. |
|                                                        | 7.Hệ thống thông báo đăng nhập thành công và chuyển hướng người dùng đến trang “Home” (hoặc trang đích đã lưu). |
| Luồng thay thế                                         | (2a) Đăng nhập bằng Google / Facebook: |
|                                                        |   2a1.Người dùng nhấn nút tương ứng. |
|                                                        |   2a2.Hệ thống thực hiện luồng xã hội (xem UC-1.3). |
|                                                        | (2b) Quên mật khẩu: |
|                                                        |   2b1.Người dùng nhấn liên kết “Forgot password”. |
|                                                        |   2b2.Hệ thống chuyển đến luồng quên và đặt lại mật khẩu (UC-1.4). |
| Luồng ngoại lệ                                         | (5a) Dữ liệu không hợp lệ: |
|                                                        |   5a1.Hệ thống hiển thị lỗi ngay dưới các trường nhập liệu. |
|                                                        |   5a2.Người dùng nhập lại thông tin. |
|                                                        | (6a) Sai email hoặc mật khẩu: Hệ thống hiển thị thông báo lỗi; trên form “Sign In” không áp dụng |
|                                                        |   cách đăng xuất toàn bộ ứng dụng như các thao tác khác. |
|                                                        | (E1) Lỗi hệ thống: Hệ thống hiển thị thông báo lỗi nội bộ hoặc lỗi kết nối. |

========================================================================================
  UC-1.2 — Đăng ký tài khoản
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.2 |
| Tên Use Case                                           | Đăng ký tài khoản |
| Mô tả                                                  | Cho phép người dùng đăng ký tài khoản mới; sau khi gửi form, hệ thống gửi mã OTP qua email để xác thực và kích hoạt |
|                                                        | tài khoản trước khi người dùng đăng nhập. |
| Đối tượng                                              | Người dùng (chưa đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng chưa có tài khoản đăng nhập hợp lệ trên trình duyệt. |
|                                                        | 2.Người dùng đang ở trang “Sign Up”. |
| Hậu điều kiện                                          | 1.Tài khoản được lưu trong cơ sở dữ liệu với trạng thái đã xác thực sau khi OTP hợp lệ. |
|                                                        | 2.Hệ thống chuyển hướng người dùng đến trang “Sign In”. |
| Luồng cơ bản                                           | 1.Người dùng truy cập vào trang “Sign Up”. |
|                                                        | 2.Hệ thống hiển thị form đăng ký; có nút đăng ký bằng Google và Facebook. |
|                                                        | 3.Người dùng nhập thông tin vào form. |
|                                                        | 4.Người dùng nhấn nút gửi đăng ký. |
|                                                        | 5.Hệ thống (Frontend) kiểm tra tính hợp lệ của dữ liệu (định dạng email, độ mạnh mật khẩu, mật khẩu khớp nhau, |
|                                                        |   các trường bắt buộc khác nếu có). |
|                                                        | 6.Hệ thống (Backend) kiểm tra email đã tồn tại trong cơ sở dữ liệu chưa. |
|                                                        | 7.Hệ thống (Backend) tạo tài khoản trạng thái chưa xác thực, sinh mã OTP và gửi email chứa OTP cho người dùng. |
|                                                        | 8.Hệ thống thông báo đăng ký thành công và hiển thị Popup (Modal) nhập mã OTP. |
|                                                        | 9.Người dùng mở email, lấy mã OTP và nhập vào Popup. |
|                                                        | 10.Người dùng nhấn “Verify”. |
|                                                        | 11.Hệ thống (Backend) kiểm tra mã OTP (đúng mã và còn trong thời hạn). |
|                                                        | 12.Hệ thống (Backend) cập nhật trạng thái tài khoản thành đã xác thực. |
|                                                        | 13.Hệ thống hiển thị thông báo thành công và chuyển hướng người dùng đến trang “Sign In”. |
| Luồng thay thế                                         | (2a) Đăng ký hoặc đăng nhập bằng Google / Facebook: |
|                                                        |   2a1.Người dùng nhấn nút tương ứng. |
|                                                        |   2a2.Hệ thống thực hiện luồng xã hội (xem UC-1.3). |
| Luồng ngoại lệ                                         | (5a) Dữ liệu không hợp lệ: |
|                                                        |   5a1.Hệ thống hiển thị lỗi ngay dưới các trường nhập liệu (ví dụ mật khẩu không khớp, chưa đủ mạnh). |
|                                                        |   5a2.Người dùng nhập lại thông tin. |
|                                                        | (6a) Email đã tồn tại: |
|                                                        |   6a1.Hệ thống hiển thị thông báo lỗi (ví dụ tài khoản đã tồn tại). |
|                                                        |   6a2.Người dùng nhập email khác hoặc chuyển sang trang “Sign In”. |
|                                                        | (11a) Mã OTP không đúng hoặc hết hạn: |
|                                                        |   11a1.Hệ thống hiển thị thông báo “Invalid OTP” hoặc “OTP has expired” (hoặc tương đương). |
|                                                        |   11a2.Người dùng nhập lại mã OTP đúng. |
|                                                        | (E1) Lỗi kết nối hoặc máy chủ không phản hồi: Hệ thống hiển thị thông báo lỗi kết nối (có thể thử máy chủ dự phòng |
|                                                        |   khi cấu hình cho phép). |
|                                                        | (E2) Lỗi hệ thống: Hệ thống hiển thị thông báo lỗi theo máy chủ hoặc thông báo chung lỗi nội bộ. |

========================================================================================
  UC-1.3 — Đăng ký hoặc đăng nhập bằng Google / Facebook
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.3 |
| Tên Use Case                                           | Đăng ký hoặc đăng nhập bằng Google / Facebook |
| Mô tả                                                  | Cho phép người dùng đăng nhập hoặc tạo tài khoản thông qua Google hoặc Facebook mà không cần |
|                                                        | nhập form email trên trang “Sign Up” / “Sign In”. |
| Đối tượng                                              | Người dùng (chưa đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng đang ở trang “Sign Up” hoặc “Sign In”. |
|                                                        | 2.Trình duyệt cho phép mở cửa sổ đăng nhập của Google/Facebook (Facebook có thể không dùng được khi trang không bảo mật). |
| Hậu điều kiện                                          | 1.Hệ thống lưu phiên đăng nhập trên trình duyệt; có thể cập nhật ngôn ngữ giao diện theo hồ sơ. |
|                                                        | 2.Từ “Sign Up”: chuyển đến “Home”. Từ “Sign In”: chuyển đến trang trước đó (nếu có) hoặc “Home”. |
| Luồng cơ bản                                           | 1.Người dùng đang ở trang “Sign Up” hoặc “Sign In”. |
|                                                        | 2.Người dùng nhấn nút đăng ký hoặc đăng nhập bằng Google hoặc Facebook. |
|                                                        | 3.Hệ thống mở luồng xác thực của nhà cung cấp và nhận xác nhận từ người dùng. |
|                                                        | 4.Hệ thống (Frontend) gửi token xác thực lên máy chủ EngSocial. |
|                                                        | 5.Hệ thống (Backend) xử lý tài khoản và trả về phiên đăng nhập. |
|                                                        | 6.Hệ thống lưu phiên trên trình duyệt và chuyển hướng người dùng theo quy tắc ở hậu điều kiện. |
| Luồng thay thế                                         | Không có. |
| Luồng ngoại lệ                                         | (3a) Người dùng đóng cửa sổ hoặc nhà cung cấp báo lỗi: |
|                                                        |   3a1.Hệ thống hiển thị thông báo lỗi trên trang. |
|                                                        |   3a2.Người dùng thử lại. |
|                                                        | (5a) Máy chủ từ chối: Hệ thống hiển thị thông báo lỗi hoặc thông báo đăng nhập không thành công. |
|                                                        | (E1) Lỗi hệ thống: Hệ thống hiển thị thông báo lỗi nội bộ hoặc lỗi kết nối. |

========================================================================================
  UC-1.4 — Quên và đặt lại mật khẩu
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.4 |
| Tên Use Case                                           | Quên và đặt lại mật khẩu |
| Mô tả                                                  | Cho phép người dùng yêu cầu gửi email để đặt lại mật khẩu, sau đó nhập mật khẩu mới thông qua liên kết kèm mã |
|                                                        | trên email. Nội dung và việc gửi email do máy chủ xử lý. |
| Đối tượng                                              | Người dùng (chưa đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng chưa đăng nhập. |
|                                                        | 2.Người dùng truy cập trang quên mật khẩu (có email tài khoản cần khôi phục) hoặc mở liên kết đặt lại từ email. |
| Hậu điều kiện                                          | 1.Sau khi gửi yêu cầu thành công: Hệ thống hiển thị thông báo đã gửi (theo ngôn ngữ giao diện). |
|                                                        | 2.Sau khi đặt lại thành công: Mật khẩu được cập nhật trên máy chủ và hệ thống chuyển hướng người dùng đến trang “Sign In”. |
| Luồng cơ bản                                           | 1.Người dùng truy cập trang quên mật khẩu. |
|                                                        | 2.Hệ thống hiển thị form gồm ô Email và nút gửi. |
|                                                        | 3.Người dùng nhập email và nhấn gửi. |
|                                                        | 4.Hệ thống (Frontend) kiểm tra định dạng email. |
|                                                        | 5.Hệ thống (Frontend) gửi yêu cầu quên mật khẩu đến máy chủ. |
|                                                        | 6.Hệ thống (Backend) xử lý yêu cầu và gửi email chứa liên kết đặt lại mật khẩu (theo cấu hình máy chủ). |
|                                                        | 7.Hệ thống hiển thị thông báo đã gửi yêu cầu (theo ngôn ngữ giao diện). |
|                                                        | 8.Người dùng mở liên kết trong email. |
|                                                        | 9.Hệ thống đọc mã trên địa chỉ; nếu có mã, hiển thị form mật khẩu mới và xác nhận mật khẩu. |
|                                                        | 10.Người dùng nhập mật khẩu mới hai lần và nhấn xác nhận. |
|                                                        | 11.Hệ thống (Frontend) kiểm tra độ mạnh và khớp hai lần nhập. |
|                                                        | 12.Hệ thống (Backend) kiểm tra mã và cập nhật mật khẩu. |
|                                                        | 13.Hệ thống hiển thị thông báo thành công và chuyển hướng người dùng đến trang “Sign In”. |
| Luồng thay thế                                         | (8a) Người dùng mở trực tiếp liên kết đặt lại từ email (chưa qua bước 1–7): |
|                                                        |   8a1.Hệ thống thực hiện từ bước 9 của luồng cơ bản. |
| Luồng ngoại lệ                                         | (4a) Email không hợp lệ (khi gửi yêu cầu): |
|                                                        |   4a1.Hệ thống hiển thị lỗi dưới ô email. |
|                                                        |   4a2.Người dùng sửa và gửi lại. |
|                                                        | (5a) Máy chủ hoặc mạng lỗi khi gửi yêu cầu: Hệ thống hiển thị thông báo lỗi. |
|                                                        | (9a) Không có mã trên liên kết đặt lại: |
|                                                        |   9a1.Hệ thống hiển thị hướng dẫn và cho phép quay lại trang quên mật khẩu (từ bước 1). |
|                                                        | (11a) Dữ liệu mật khẩu không hợp lệ: |
|                                                        |   11a1.Hệ thống hiển thị lỗi dưới các trường nhập liệu. |
|                                                        |   11a2.Người dùng nhập lại thông tin. |
|                                                        | (12a) Mã không đúng hoặc hết hạn: Hệ thống hiển thị thông báo lỗi. |
|                                                        | (E1) Lỗi hệ thống: Hệ thống hiển thị thông báo lỗi nội bộ hoặc lỗi kết nối. |

========================================================================================
  UC-1.6 — Hồ sơ người dùng
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.6 |
| Tên Use Case                                           | Xem và cập nhật hồ sơ |
| Mô tả                                                  | Cho phép người dùng đã đăng nhập xem hồ sơ (của mình hoặc của người khác theo định danh), chỉnh sửa thông tin, |
|                                                        | tải ảnh đại diện, đổi mật khẩu; tuỳ chọn tài khoản (ngôn ngữ, chủ đề, thông báo, …) đồng bộ qua máy chủ xác thực khi người dùng lưu trên màn hình cài đặt. |
| Đối tượng                                              | Người dùng (đã đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng đã đăng nhập. |
|                                                        | 2.Người dùng truy cập trang hồ sơ hoặc chức năng chỉnh sửa. |
| Hậu điều kiện                                          | 1.Thông tin hồ sơ hiển thị đúng theo phản hồi thành công từ máy chủ. |
|                                                        | 2.Sau khi lưu, dữ liệu trên giao diện và bản lưu cục bộ (nếu có) phản ánh thay đổi. |
| Luồng cơ bản                                           | 1.Người dùng mở trang hồ sơ. |
|                                                        | 2.Hệ thống (Frontend) gửi yêu cầu lấy thông tin hồ sơ (và thông tin tài khoản hiện tại nếu cần) đến máy chủ. |
|                                                        | 3.Hệ thống hiển thị dữ liệu hồ sơ. |
|                                                        | 4.Người dùng chỉnh sửa thông tin hoặc tuỳ chọn và nhấn lưu. |
|                                                        | 5.Hệ thống (Frontend) kiểm tra tính hợp lệ của dữ liệu (nếu có). |
|                                                        | 6.Hệ thống (Frontend) gửi yêu cầu cập nhật đến máy chủ (hồ sơ, ảnh, mật khẩu hoặc tuỳ chọn tài khoản tuỳ thao tác). |
|                                                        | 7.Hệ thống (Backend) lưu thay đổi và trả kết quả. |
|                                                        | 8.(Tuỳ chọn) Người dùng đổi ảnh đại diện: Hệ thống (Frontend) gửi tệp lên máy chủ; Hệ thống cập nhật ảnh hiển thị. |
|                                                        | 9.(Tuỳ chọn) Người dùng đổi mật khẩu: Hệ thống (Frontend) gửi yêu cầu đổi mật khẩu đến máy chủ; Hệ thống hiển thị kết quả. |
|                                                        | 10.Hệ thống hiển thị thông báo thành công hoặc lỗi. |
| Luồng thay thế                                         | Không có. |
| Luồng ngoại lệ                                         | (2a) Không lấy được hồ sơ: Hệ thống hiển thị thông báo lỗi. |
|                                                        | (5a) Dữ liệu không hợp lệ: |
|                                                        |   5a1.Hệ thống hiển thị lỗi ngay dưới các trường nhập liệu. |
|                                                        |   5a2.Người dùng nhập lại thông tin. |
|                                                        | (7a) Máy chủ từ chối cập nhật: Hệ thống hiển thị thông báo lỗi theo máy chủ. |
|                                                        | (E1) Lỗi hệ thống hoặc mạng: Hệ thống hiển thị thông báo lỗi nội bộ hoặc lỗi kết nối. |

========================================================================================
  UC-1.7 — Thông báo

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.7 |
| Tên Use Case                                           | Xem và xử lý thông báo |
| Mô tả                                                  | Cho phép người dùng đã đăng nhập xem danh sách thông báo hệ thống, biết số thông báo chưa đọc, đánh dấu đã đọc |
|                                                        | (từng mục hoặc toàn bộ) hoặc xóa thông báo; mọi dữ liệu đồng bộ qua máy chủ thông báo. |
| Đối tượng                                              | Người dùng (đã đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng đã đăng nhập. |
|                                                        | 2.Người dùng mở trang hoặc vùng chức năng thông báo (kèm biểu tượng số lượng chưa đọc nếu có). |
| Hậu điều kiện                                          | 1.Danh sách thông báo và số chưa đọc hiển thị đúng theo phản hồi thành công từ máy chủ. |
|                                                        | 2.Sau khi đánh dấu đã đọc hoặc xóa, giao diện phản ánh trạng thái mà máy chủ xác nhận. |
| Luồng cơ bản                                           | 1.Người dùng mở danh sách thông báo. |
|                                                        | 2.Hệ thống (Frontend) gửi yêu cầu lấy danh sách thông báo đến máy chủ (có thể kèm phân trang). |
|                                                        | 3.Hệ thống (Backend) trả về danh sách; Hệ thống hiển thị cho người dùng. |
|                                                        | 4.(Tuỳ giao diện) Hệ thống (Frontend) gửi yêu cầu lấy số thông báo chưa đọc đến máy chủ để cập nhật badge. |
|                                                        | 5.Người dùng mở một thông báo hoặc chọn thao tác “đánh dấu đã đọc”, “đánh dấu đã đọc tất cả” hoặc “xóa”. |
|                                                        | 6.Hệ thống (Frontend) gửi yêu cầu cập nhật hoặc xóa tương ứng đến máy chủ. |
|                                                        | 7.Hệ thống (Backend) xử lý và trả kết quả; Hệ thống làm mới danh sách và số chưa đọc trên giao diện. |
| Luồng thay thế                                         | Không có. |
| Luồng ngoại lệ                                         | (2a) Không lấy được danh sách thông báo: |
|                                                        |   2a1.Hệ thống hiển thị thông báo lỗi. |
|                                                        |   2a2.Người dùng thử tải lại hoặc quay lại sau. |
|                                                        | (6a) Máy chủ từ chối cập nhật hoặc xóa: Hệ thống hiển thị thông báo lỗi theo phản hồi máy chủ. |
|                                                        | (E1) Lỗi hệ thống hoặc mạng: Hệ thống hiển thị thông báo lỗi nội bộ hoặc lỗi kết nối; nếu máy chủ trả 401 thì xử lý |
|                                                        |   theo luồng chung của ứng dụng (ví dụ yêu cầu đăng nhập lại). |

  UC-1.12 — Xem danh sách bạn bè

| Tiền điều kiện                                         | 1.Người dùng đã đăng nhập. |
|                                                        | 2.Người dùng có quyền xem danh sách thành viên trong nhóm đang chọn (theo chính sách riêng tư nhóm). |

  UC-1.29 — Duyệt yêu cầu xin vào nhóm cộng đồng
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.29 |
| Tên Use Case                                           | Duyệt yêu cầu xin vào nhóm cộng đồng |
| Mô tả                                                  | Cho phép chủ nhóm (hoặc người có quyền) xem danh sách yêu cầu xin vào nhóm và duyệt hoặc từ chối từng yêu cầu; dữ liệu đồng bộ qua máy chủ. |
| Đối tượng                                              | Người dùng (đã đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng đã đăng nhập và có quyền quản trị nhóm (ví dụ chủ nhóm). |
|                                                        | 2.Nhóm có chế độ duyệt thành viên và tồn tại ít nhất một yêu cầu xin vào đang chờ (tuỳ thời điểm). |
| Hậu điều kiện                                          | Danh sách yêu cầu và trạng thái thành viên nhóm cập nhật đúng theo phản hồi thành công từ máy chủ. |
| Luồng cơ bản                                           | 1.Người dùng có quyền quản trị mở màn hình “Yêu cầu xin vào” / hàng chờ duyệt của nhóm từ trang chi tiết nhóm hoặc menu quản trị. |
|                                                        | 2.Hệ thống (Frontend) gửi yêu cầu lấy danh sách yêu cầu xin vào đang chờ đến máy chủ. |

|                                                        | 4.Hệ thống hiển thị danh sách cho người chủ nhóm (hoặc người có quyền). |
|                                                        | 5.Người dùng xem thông tin người xin vào và chọn “Duyệt” hoặc “Từ chối” cho một mục. |
|                                                        | 6.Hệ thống (Frontend) gửi yêu cầu xử lý yêu cầu xin vào (duyệt hoặc từ chối) kèm định danh yêu cầu đến máy chủ. |
|                                                        | 7.Hệ thống (Backend) cập nhật trạng thái yêu cầu và thành viên nhóm (nếu duyệt), trả kết quả. |
|                                                        | 8.Hệ thống làm mới danh sách yêu cầu trên giao diện và (tuỳ chính sách) ghi nhận thông báo cho người được duyệt/từ chối. |
|                                                        | 9.(Tuỳ thiết kế) Người dùng lặp lại bước 5–8 cho yêu cầu khác hoặc tải thêm trang danh sách. |
