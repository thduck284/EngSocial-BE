========================================================================================
  UC-1.31 — Xem chi tiết bài viết
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.31 |
| Tên Use Case                                           | Xem chi tiết bài viết |
| Mô tả                                                  | Cho phép người dùng đã đăng nhập xem nội dung chi tiết một bài viết theo định danh; dữ liệu do máy chủ trả về. |
| Đối tượng                                              | Người dùng (đã đăng nhập nếu thao tác ghi) |
| Tiền điều kiện                                         | 1.Người dùng đã đăng nhập khi chính sách yêu cầu phiên đăng nhập để xem bài. |
|                                                        | 2.Người dùng có định danh bài viết (từ bảng tin, liên kết chia sẻ, thông báo, …). |
| Hậu điều kiện                                          | Nội dung chi tiết bài viết (tiêu đề, nội dung, đính kèm, số tương tác — tuỳ thiết kế) hiển thị đúng theo phản hồi thành công từ máy chủ. |
| Luồng cơ bản                                           | 1.Người dùng chọn một bài viết để xem chi tiết (từ danh sách bảng tin hoặc liên kết). |
|                                                        | 2.Hệ thống (Frontend) gửi yêu cầu lấy chi tiết bài viết theo định danh đến máy chủ. |
|                                                        | 3.Hệ thống (Backend) trả về dữ liệu bài viết. |
|                                                        | 4.Hệ thống hiển thị trang chi tiết (nội dung, tác giả, thời gian, …). |
|                                                        | 5.(Tuỳ thiết kế) Người dùng kéo làm mới hoặc mở lại cùng bài; Hệ thống (Frontend) gửi yêu cầu lấy lại nội dung nếu cần. |
|                                                        | 6.Hệ thống (Backend) trả về dữ liệu mới; Hệ thống cập nhật giao diện. |
| Luồng thay thế                                         | Không có. |
| Luồng ngoại lệ                                         | (2a) Không lấy được chi tiết (bài không tồn tại, không có quyền xem, …): Hệ thống hiển thị thông báo theo máy chủ. |
|                                                        | (5a) Làm mới thất bại: Hệ thống hiển thị thông báo lỗi; nội dung đã hiển thị có thể giữ nguyên. |
|                                                        | (E1) Lỗi hệ thống hoặc mạng: Hệ thống hiển thị thông báo lỗi nội bộ hoặc lỗi kết nối; nếu máy chủ trả 401 thì xử lý |
|                                                        |   theo luồng chung của ứng dụng (ví dụ yêu cầu đăng nhập lại). |

  UC-1.32 — Đăng bài viết
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.32 |
| Tên Use Case                                           | Đăng bài viết |
| Mô tả                                                  | Cho phép người dùng đã đăng nhập tạo bài viết mới trên bảng tin với nội dung do giao diện thu thập; máy chủ lưu và trả về bài đã đăng. |
| Đối tượng                                              | Người dùng (đã đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng đã đăng nhập. |
|                                                        | 2.Người dùng mở biểu mẫu đăng bài (nút “Tạo bài”, FAB, … tuỳ thiết kế). |
| Hậu điều kiện                                          | Bài viết được ghi nhận trên máy chủ; giao diện chuyển tới chi tiết bài mới hoặc cập nhật bảng tin theo phản hồi thành công. |
| Luồng cơ bản                                           | 1.Người dùng nhập tiêu đề, nội dung và (tuỳ chọn) đính kèm ảnh/tệp theo biểu mẫu. |
|                                                        | 2.Người dùng chọn “Đăng” / tương đương. |
|                                                        | 3.Hệ thống (Frontend) kiểm tra hợp lệ cục bộ (nếu có); nếu hợp lệ, gửi yêu cầu tạo bài viết kèm dữ liệu biểu mẫu đến máy chủ. |
|                                                        | 4.Hệ thống (Backend) lưu bài và trả về định danh bài cùng dữ liệu hiển thị. |
|                                                        | 5.Hệ thống hiển thị xác nhận (tuỳ thiết kế) và điều hướng tới trang chi tiết bài hoặc về bảng tin đã làm mới. |
| Luồng thay thế                                         | Không có. |
| Luồng ngoại lệ                                         | (3a) Dữ liệu không hợp lệ trên giao diện (thiếu trường bắt buộc, vượt độ dài, …): Hệ thống hiển thị gợi ý trước khi gửi máy chủ. |
|                                                        | (4a) Đăng bài thất bại trên máy chủ (spam, lỗi upload, …): Hệ thống hiển thị thông báo theo máy chủ. |
|                                                        | (E1) Lỗi hệ thống hoặc mạng; 401: xử lý theo luồng chung ứng dụng. |

Các chức năng mở rộng (chuyển tiếp, sửa/thu hồi tin, cảm xúc trên tin, tạo nhóm, cài đặt thông báo theo hội thoại) tuỳ EngSocial-FE nhưng cùng nguyên tắc gọi máy chủ. |

|                                                        | 3.Sau khi gửi tin thành công, tin xuất hiện trong luồng hội thoại và đối phương có thể nhận theo cơ chế realtime/polling của hệ thống. |
| Luồng cơ bản                                           | 1.Người dùng mở danh sách hội thoại. |

|                                                        | 3.Hệ thống (Backend) trả dữ liệu; Hệ thống hiển thị danh sách và huy hiệu chưa đọc (nếu có). |
|                                                        | 4.Người dùng chọn một hội thoại. |
|                                                        | 5.Hệ thống (Frontend) gửi yêu cầu lấy lịch sử tin nhắn của hội thoại đó đến máy chủ (có thể phân trang). |
|                                                        | 6.Hệ thống (Backend) trả về tin nhắn; Hệ thống hiển thị luồng hội thoại. |

|                                                        | 10.Hệ thống cập nhật luồng tin, hàng đợi gửi/nhận và trạng thái đã đọc trên giao diện. |

|                                                        | (1b) Người dùng tạo nhóm chat (nếu sản phẩm hỗ trợ): Hệ thống (Frontend) gửi yêu cầu tạo nhóm đến máy chủ; sau khi thành công, Hệ thống mở hội thoại nhóm mới. |
| Luồng ngoại lệ                                         | (2a) Không lấy được danh sách hội thoại: |
|                                                        |   2a1.Hệ thống hiển thị thông báo lỗi. |
|                                                        |   2a2.Người dùng thử tải lại. |
|                                                        | (5a) Không lấy được tin nhắn: Hệ thống hiển thị thông báo lỗi. |
|                                                        | (7a) Đánh dấu đã đọc thất bại: Hệ thống vẫn hiển thị tin nhưng có thể giữ huy hiệu chưa đọc; người dùng thử lại sau. |
|                                                        | (9a) Gửi tin hoặc tệp thất bại (dung lượng, định dạng, quyền, …): Hệ thống hiển thị thông báo theo máy chủ. |
