========================================================================================
  UC-1.63 — Xem danh sách nhóm học tập
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.63 |
| Tên Use Case                                           | Xem danh sách nhóm học tập |
| Mô tả                                                  | Cho phép người dùng đã đăng nhập xem các nhóm chat gắn với khu nhóm học tập trên |
|                                                        | Dashboard (tiêu đề theo bản dịch `dashboard.studyGroups`), mở modal danh sách đầy đủ nếu có, |
|                                                        | và chọn một nhóm để chuyển sang màn hình Tin nhắn với cuộc trò chuyện tương ứng. |
| Đối tượng                                              | Người dùng (đã đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng đã đăng nhập. |
|                                                        | 2.Người dùng mở trang Dashboard (hoặc trang tích hợp cùng hook tải danh sách nhóm chat). |
| Hậu điều kiện                                          | 1.Danh sách nhóm chat học tập hiển thị đúng theo phản hồi thành công từ máy chủ (sau khi lọc các nhóm chat). |
|                                                        | 2.Khi người dùng chọn một nhóm, ứng dụng điều hướng tới luồng Tin nhắn đúng mã cuộc trò chuyện. |
| Luồng cơ bản                                           | 1.Người dùng mở Dashboard. |
|                                                        | 2.Hệ thống (Frontend) gửi yêu cầu lấy danh sách chat đến máy chủ. |
|                                                        | 3.Hệ thống (Backend) trả về danh sách chat. |
|                                                        | 4.Hệ thống (Frontend) lọc các mục là nhóm chat; Hệ thống hiển thị thẻ nhóm (tên, ảnh, số thành viên, trạng thái online — tuỳ thiết kế). |
|                                                        | 5.(Tuỳ thiết kế) Người dùng nhấn tiêu đề khu nhóm học tập để mở modal danh sách đầy đủ; Hệ thống hiển thị cùng nguồn dữ liệu đã lọc. |
|                                                        | 6.Người dùng chọn một nhóm (từ sidebar hoặc trong modal). |
|                                                        | 7.Hệ thống (Frontend) điều hướng người dùng đến trang Tin nhắn với đường dẫn cuộc trò chuyện tương ứng. |
|                                                        | 8.(Tuỳ thiết kế) Người dùng kéo làm mới hoặc quay lại Dashboard; Hệ thống (Frontend) gửi lại yêu cầu lấy danh sách đến máy chủ. |
|                                                        | 9.Hệ thống (Backend) trả về dữ liệu mới; Hệ thống cập nhật danh sách trên giao diện. |
| Luồng thay thế                                         | Không có. |
| Luồng ngoại lệ                                         | (2a) Không tải được danh sách chat lần đầu: |
|                                                        |   2a1.Hệ thống hiển thị thông báo lỗi hoặc danh sách rỗng theo chính sách giao diện. |
|                                                        |   2a2.Người dùng thử tải lại trang hoặc thử lại sau. |
|                                                        | (8a) Làm mới thất bại: Hệ thống hiển thị thông báo lỗi; dữ liệu đã hiển thị có thể giữ nguyên. |
|                                                        | (E1) Lỗi hệ thống hoặc mạng; 401: xử lý theo luồng chung ứng dụng. |

========================================================================================
  UC-1.65 — Tìm chat hoặc bạn bè rồi mở cuộc trò chuyện
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.65 |
| Tên Use Case                                           | Tìm chat hoặc bạn bè rồi mở cuộc trò chuyện |
| Mô tả                                                  | Cho phép người dùng đã đăng nhập dùng ô tìm kiếm trên trang Tin nhắn để lọc danh sách chat cục bộ |
|                                                        | hoặc xem gợi ý bạn bè; khi chọn một bạn, hệ thống tạo hoặc lấy chat riêng một-một và mở luồng tin nhắn. |
| Đối tượng                                              | Người dùng (đã đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng đã đăng nhập. |
|                                                        | 2.Người dùng đang ở trang Tin nhắn. |
| Hậu điều kiện                                          | 1.Cuộc trò chuyện được chọn hiển thị với luồng tin tương ứng. |
|                                                        | 2.Địa chỉ hoặc tham số điều hướng (ví dụ tham số `with`) phản ánh cuộc trò chuyện đang mở — tuỳ thiết kế ứng dụng. |
| Luồng cơ bản                                           | 1.Người dùng nhập từ khóa vào ô tìm kiếm chat / bạn bè. |
|                                                        | 2.Hệ thống (Frontend) lọc danh sách chat đang có hoặc gửi yêu cầu tìm bạn đến máy chủ (tuỳ triển khai). |
|                                                        | 3.Hệ thống hiển thị kết quả khớp (danh sách chat hoặc mục bạn bè). |
|                                                        | 4.Người dùng chọn một cuộc trò chuyện trong kết quả. |
|                                                        | 5.Hệ thống (Frontend) điều hướng tới luồng chat đã chọn. |
|                                                        | 6.(Nhánh bạn bè) Người dùng chọn một bạn từ khu gợi ý: Hệ thống (Frontend) gửi yêu cầu lấy hoặc tạo chat riêng một-một với người đó đến máy chủ. |
|                                                        | 7.Hệ thống (Backend) trả về thông tin cuộc trò chuyện; Hệ thống mở luồng tin nhắn. |
| Luồng thay thế                                         | (4a) Người dùng chỉ chọn cuộc trò chuyện đã có sẵn trong danh sách (không qua gợi ý bạn): thực hiện từ bước 5. |
| Luồng ngoại lệ                                         | (2a) Không có kết quả tìm: Hệ thống hiển thị trạng thái rỗng; người dùng chỉnh lại từ khóa. |
|                                                        | (6a) Không tạo hoặc không mở được cuộc trò chuyện: Hệ thống hiển thị thông báo theo máy chủ. |
|                                                        | (E1) Lỗi hệ thống hoặc mạng; 401: xử lý theo luồng chung ứng dụng. |

========================================================================================
  UC-1.67 — Mở nhóm chat và xem lịch sử tin nhắn
========================================================================================

| Thuộc tính                                             | Nội dung |
|--------------------------------------------------------|--------------------------------------------------------------------------------|
| Mã Use Case                                            | UC-1.67 |
| Tên Use Case                                           | Mở nhóm chat và xem lịch sử tin nhắn |
| Mô tả                                                  | Cho phép người dùng đã đăng nhập mở một nhóm chat, tải lịch sử tin nhắn (có phân trang theo con trỏ tin cũ), |
|                                                        | đánh dấu đã đọc khi xem, và tải thêm tin cũ khi cuộn — đồng bộ qua máy chủ. |
| Đối tượng                                              | Người dùng (đã đăng nhập) |
| Tiền điều kiện                                         | 1.Người dùng đã đăng nhập và là thành viên của nhóm chat. |
|                                                        | 2.Người dùng đã chọn nhóm chat từ danh sách (hoặc sau khi tạo nhóm thành công). |
| Hậu điều kiện                                          | 1.Luồng tin nhắn hiển thị đúng theo phản hồi thành công từ máy chủ. |
|                                                        | 2.Trạng thái đã đọc / huy hiệu chưa đọc phản ánh sau khi máy chủ xác nhận (tuỳ chính sách). |
| Luồng cơ bản                                           | 1.Người dùng chọn một nhóm chat trong danh sách bên trái. |
|                                                        | 2.Hệ thống (Frontend) hiển thị trạng thái đang tải luồng chat (nếu có). |
|                                                        | 3.Hệ thống (Frontend) gửi yêu cầu lấy danh sách tin nhắn (trang đầu, kèm giới hạn số tin) đến máy chủ. |
|                                                        | 4.Hệ thống (Backend) trả về danh sách tin nhắn. |
|                                                        | 5.Hệ thống hiển thị luồng tin trong cuộc trò chuyện. |
|                                                        | 6.Hệ thống (Frontend) gửi yêu cầu đánh dấu đã đọc khi người dùng đã xem luồng chat (tuỳ chính sách sản phẩm). |
|                                                        | 7.(Tuỳ thiết kế) Người dùng cuộn lên để xem tin cũ hơn; Hệ thống (Frontend) gửi yêu cầu lấy thêm tin với con trỏ phân trang. |
|                                                        | 8.Hệ thống (Backend) trả về thêm tin; Hệ thống nối tin vào luồng theo quy ước giao diện. |
| Luồng thay thế                                         | Không có. |
| Luồng ngoại lệ                                         | (3a) Không tải được lịch sử tin lần đầu (không có quyền, cuộc trò chuyện không tồn tại, …): Hệ thống hiển thị thông báo theo máy chủ. |
|                                                        | (7a) Tải thêm tin cũ thất bại: Hệ thống hiển thị thông báo lỗi; các tin đã có giữ nguyên. |
|                                                        | (6a) Đánh dấu đã đọc thất bại: Hệ thống vẫn hiển thị tin; huy hiệu chưa đọc có thể giữ nguyên. |
|                                                        | (E1) Lỗi hệ thống hoặc mạng; 401: xử lý theo luồng chung ứng dụng. |

|                                                        | 5.Hệ thống (Frontend) gửi yêu cầu gửi tin đến máy chủ (dạng multipart nếu có tệp, hoặc JSON kèm liên kết đính kèm nếu chỉ GIF). |

|                                                        | 7.Hệ thống cập nhật luồng chat và (tuỳ) cập nhật danh sách chat; các thành viên khác nhận tin theo cơ chế realtime hoặc làm mới — tuỳ triển khai. |

| Luồng ngoại lệ                                         | (5a) Gửi tin thất bại (dung lượng, định dạng, bị chặn, …): Hệ thống hiển thị thông báo theo máy chủ. |
|                                                        | (E1) Lỗi hệ thống hoặc mạng; 401: xử lý theo luồng chung ứng dụng. |

| Tên Use Case                                           | Chỉnh sửa tin nhắn |

|                                                        | 4.Người dùng xác nhận gửi bản cập nhật. |
|                                                        | 5.Hệ thống (Frontend) gửi yêu cầu cập nhật tin nhắn đến máy chủ. |
|                                                        | 6.Hệ thống (Backend) lưu và trả về tin đã cập nhật. |
